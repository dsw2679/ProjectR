## 1. 세이브 / 로드 시스템

### 1.1 세이브 슬롯

세이브 슬롯은 육성 캐릭터 단위. 최대 3캐릭터까지 제공.

### 1.2 자동 저장 트리거

자동 저장은 아래 이벤트 발생 시 즉시 실행된다.

| 트리거 이벤트 | 설명 |
| --- | --- |
| 체크포인트 활성화 | 체크포인트 상호작용 완료 시 |
| 거점 귀환 | 포탈을 통해 거점으로 복귀 시 |
| 지역 이동 | 전투 지역 씬 전환 완료 시 |
| 보스 처치 | 보스 사망 처리 완료 시 |

### 1.3 저장 처리 흐름

1. 자동 저장 트리거 이벤트 발생
2. 화면 우상단에 저장 중 인디케이터 표시 (저장 아이콘 애니메이션)
3. 현재 게임 상태를 게임 상태 데이터로 변환
4. 디스크에 저장
5. 저장 완료 후 인디케이터 소멸

### 1.4 로드 처리 흐름

1. 메인 메뉴 "계속하기" 또는 슬롯 선택 후 로드 실행
2. 디스크에서 저장 데이터 불러오기
3. 저장된 레벨로 씬 전환
4. 플레이어 상태(체력, 탄약, 인벤토리) 복원
5. 체크포인트 활성 상태 복원
6. 플레이어 스폰 위치: 마지막 활성 체크포인트 위치

## 3. 멀티플레이 진행도 처리

### 3.1 기본 원칙

- 세계 진행도(레벨 클리어, 보스 처치, 체크포인트)는 호스트 세이브 데이터 기준으로 적용
- 클라이언트 참가 플레이어는 게스트로 참여하며 세계 진행도는 호스트에게만 반영

### 3.2 클라이언트 개인 진행도

- 멀티플레이 세션 참여 중 획득한 아이템(무기, Mod, 재료)은 클라이언트 개인 세이브 데이터에 반영
- 세션 종료(정상 퇴장) 시 클라이언트 개인 세이브 데이터 저장
- 클라이언트 비정상 연결 해제 시 세이브 시점 기준 이전 상태 유지 (TBD)

### 3.3 멀티플레이 세션 종료 후 개인 진행도 반영 기준

| 항목 | 처리 |
| --- | --- |
| 획득 아이템 | 세션 종료 시 클라이언트 세이브에 반영 |
| 호스트 세계의 체크포인트 / 클리어 상태 | 미반영 (클라이언트는 자신의 세계 진행 기준 유지) |
| 탄약, 소비 아이템 사용량 | 세션 중 소모된 수량 클라이언트 세이브에 반영 |
