## 1. 사격 시스템

### 1.1 발사 방식 분기

무기 타입에 따라 히트스캔 또는 투사체 방식으로 분기된다.

| 무기 | 발사 방식 |
| --- | --- |
| 돌격소총 | 히트스캔(Hitscan) |
| 볼트액션 | 히트스캔(Hitscan) |
| 유탄발사기 | 투사체(Projectile) |

- 히트스캔: 발사 즉시 카메라 전방으로 판정선을 실행하여 충돌 지점 계산
- 투사체: 발사 시 물리 시뮬레이션 기반 발사체를 생성

### 1.2 연사 및 단발 모드

| 무기 | 발사 모드 | 발사 속도 |
| --- | --- | --- |
| 돌격소총 | 풀오토(Full-Auto) | 600 RPM (연사 딜레이 0.1 초) |
| 유탄발사기 | 단발(Semi-Auto) | 발당 1.5 초 딜레이 |
| 볼트액션 | 볼트액션(단발) | 발당 1.8 초 딜레이 (볼트 모션 포함) |

- 단발 사격 도중 입력 버퍼를 통해 연이은 단발 사격 조작감 개선

### 1.3 탄창 및 재장전

| 무기 | 탄창 용량 | 재장전 시간 | 자동 재장전 |
| --- | --- | --- | --- |
| 돌격소총 | 30 발 | 2.0 초 | 탄창 소진 시 자동 |
| 유탄발사기 | 6 발 | 3.0 초 (전탄 재장전) | 탄창 소진 시 자동 |
| 볼트액션 | 5 발 | 2.5 초 | 탄창 소진 시 자동 |

- 탄창 소진 상태에서 사격 입력 시: 짧은 딸깍 격발음 재생 → 자동 재장전 루틴 즉시 진입
- 재장전 모션 시작 후 전체 모션의 80% 진행 전까지는 취소 불가
- 재장전 완료 후 탄창은 보유 탄약에서 차감하여 채움
- 보유 탄약이 탄창 용량보다 적을 경우 보유량 전부를 탄창에 충전

### 1.4 반동(Recoil) 개요

발사 시 크로스헤어 위치 및 카메라 상하 방향에 반동 패턴 적용. 세부 수치는 무기 장비 기획서 참조.

## 2. 피격 및 데미지 시스템

### 2.1 피해량 계산 공식

```
최종 피해량 = 무기 기본 데미지 × 부위 배율 × 상태 보정 배율
```

### 2.2 부위별 피해 배율

| 부위 | 배율 | 적용 대상 |
| --- | --- | --- |
| 일반 | 1.0x | 몸통 |
| 약점 | 1.0x + 무기별 약점 보너스 비율 | 머리 |
| 단단(TBD) | 1.0x − 부위 피해 감소 비율 | 다리, 팔 |

- 보스 약점 부위는 노출 타이밍에만 판정 활성화. 배율은 보스 전용 데이터 테이블에서 정의

### 2.3 상태 보정 배율

| 상태 | 추가 배율 |
| --- | --- |
| 그로기(Groggy) 상태 | +1.5x (전체 피해에 적용) |
| 무적 프레임 | 0x (피해 무효) |
| 기본 | 1.0x |

### 2.4 피격 리액션

- 피격 애니메이션 재생. 피격 방향에 따라 전/후/좌/우 분기
- 넉백: 피격 방향 반대로 일정 거리만큼 캐릭터를 밀어냄 (기본값 50 cm)
- 무적 프레임 중 피격 시 리액션 없이 피해만 무효

## 3. 투사체 시스템

### 3.1 투사체 생성

- 발사 시점에 총구 위치에 투사체 생성
- 생성과 동시에 발사 방향 및 초기 속도 부여

### 3.2 이동 방식

| 투사체 유형 | 이동 방식 | 중력 적용 |
| --- | --- | --- |
| 유탄발사기 기본 | 직사(직선 이동) | 없음 |
| 유탄발사기 특수 Mod (곡사) | 포물선(물리 시뮬레이션) | 있음 |

### 3.3 충돌 및 폭발 처리

| 항목 | 값 |
| --- | --- |
| 폭발 반경 | 300 cm (기본) |
| 직격 피해 | 무기 기본 데미지 |
| 폭발 범위 피해 | 직격 데미지의 60%, 거리에 따른 선형 감쇠 |
| 폭발 피해 감쇠 | 중심(0cm): 100%, 폭발 반경 끝: 30% |

### 3.4 도탄(Bounce) 기능 처리

**미션 가이드 필수 요구사항**에 따라 투사체 도탄 기능을 지원한다.

| 항목 | 값 |
| --- | --- |
| 최대 도탄 횟수 | 2 회 |
| 도탄 후 폭발 조건 | 적과 충돌 또는 마지막 도탄 후 2 초 경과 |
| 도탄 방향 연산 | 물리 소재(Physical Material) 기반 반사 벡터 연산 |
| 멀티플레이 동기화 타협 | 완벽한 도탄 물리 동기화 대신, 서버가 튕긴 궤적(위치)을 계산하여 클라이언트에 강제 보정(Interpolation)하는 방식을 채택하여 디싱크를 최소화함 |

### 3.5 지속형 장판 생성

특정 Mod 효과로 투사체 충돌 지점에 장판 생성. 상세 사항은 전투_게이지_상태 기획서 4절 참조.

## 4. 히트박스 및 콜리전 구성

### 4.1 판정 채널 구성

| 대상 | 역할 |
| --- | --- |
| 플레이어 피격 판정 | 플레이어 전체 캡슐 단위 |
| 일반 적 피격 판정 | 적 부위별 히트박스 |
| 투사체 | 투사체 충돌 판정 |
| 환경 | 월드 오브젝트 충돌 |

### 4.2 플레이어 히트박스

- 캡슐 형태 단일 히트박스 사용
- 구르기 무적 프레임 구간 중 플레이어 피격 판정 비활성화

### 4.3 적 부위별 히트박스

- 스켈레탈 메시 기반으로 부위별 히트박스 구성
- 부위마다 피해 배율 정의

| 부위 | 배율 |
| --- | --- |
| 머리 | 2.5x |
| 몸통 | 1.0x |
| 사지 | 0.75x |
| 약점 | 별도 정의 |

### 4.4 히트스캔 판정선

- 카메라 전방으로 단일 판정선 실행
- 최대 사거리: 5000 cm (50 m)
- 관통 없음 (첫 번째 충돌 결과만 처리)
- 판정 대상: 적 히트박스, 환경 오브젝트
