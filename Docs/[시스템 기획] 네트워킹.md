## 1. 세션 관리

### 1.1 세션 생성

- 호스트 플레이어가 메인 메뉴 또는 로비에서 세션 생성
- 세션 공개 범위 선택: 공개 / 비공개 / 친구 초대 전용

| 항목 | 값 |
| --- | --- |
| 최대 인원 | 3 명 |
| 세션 이름 | 플레이어 이름 기반 자동 생성 또는 사용자 입력 |

### 1.2 세션 탐색 및 참가

- 공개 세션 목록 조회: 세션 이름, 현재 인원 / 최대 인원, 호스트 이름 표시
- 세션 ID 또는 초대 코드로 비공개 세션 직접 참가 가능

### 1.3 세션 퇴장

| 시나리오 | 처리 |
| --- | --- |
| 클라이언트 자발적 퇴장 | 해당 플레이어 제거, 세션 계속 진행 |
| 호스트 자발적 퇴장 | 세션 종료 또는 호스트 이전(Host Migration) - TBD |
| 클라이언트 연결 해제 | 8절 예외 처리 참조 |

### 1.4 세션 상태 구분

| 상태 | 설명 |
| --- | --- |
| 로비 대기 | 게임 시작 전 대기 상태 |
| 게임 진행 중 | 플레이 중인 상태 |
| 세션 종료 | 세션이 닫힌 상태 |

## 3. 플레이어 동기화

### 3.1 위치 및 회전 동기화

- 각 플레이어의 위치, 회전이 다른 플레이어 화면에 실시간으로 반영됨
- 서버가 위치 정보를 기준값으로 관리하고 모든 플레이어에게 동기화

### 3.2 클라이언트 예측 이동

- 로컬 플레이어의 이동 입력은 서버 응답을 기다리지 않고 즉시 화면에 적용
- 서버에서 보정값 수신 시 이전 위치와의 오차가 임계치(5 cm) 초과 시 위치 보정 적용

### 3.3 애니메이션 상태 동기화

- 이동 방향, 속도, 구르기, 재장전, 사격 상태가 다른 플레이어 화면에도 반영
- 각 플레이어 화면에서 해당 정보를 기반으로 애니메이션 재생

### 3.4 생존 상태 동기화

- 다운 상태 진입 / 소생 완료 / 완전 사망 이벤트는 서버에서 처리 후 모든 플레이어에게 동시 반영

## 4. 전투 동기화

### 4.1 사격 이벤트 동기화

| 단계 | 처리 주체 |
| --- | --- |
| 입력 감지 및 발사 요청 | 클라이언트 (로컬) |
| 피격 판정 및 피해 적용 | 서버 |
| 피격 이펙트 재생 | 클라이언트 (서버 결과를 받아 재생) |

- 클라이언트가 발사 방향 및 결과를 서버에 전송, 서버에서 재검증 후 최종 피해 적용

### 4.2 피해 처리 권한

- **최종 피해량 계산은 서버에서만 수행**
- 클라이언트는 피해 요청만 서버에 전달

### 4.3 투사체 동기화

- 투사체 생성은 서버에서 실행 후 모든 클라이언트 화면에 반영
- 투사체 이동은 서버 시뮬레이션 기준, 클라이언트는 보간으로 부드럽게 표시

### 4.4 그로기 게이지 동기화

- 그로기 게이지는 서버에서 관리
- 게이지 변경 시 모든 클라이언트에 반영

### 4.5 Mod 게이지 동기화

- Mod 게이지 값은 서버 관리
- 충전 이벤트 및 발동 이벤트 서버 검증 후 전 플레이어에게 동시 반영

### 4.6 무적 프레임 동기화

- 구르기 무적 판정은 서버에서 처리
- 클라이언트가 구르기 시작 시 서버에 통보 → 서버가 무적 플래그 On/Off 관리

## 5. 적 AI 동기화

### 5.1 서버 권위 AI

- **모든 AI 로직 실행은 서버 전용**
- 클라이언트에서 AI 행동 연산 없음

### 5.2 AI 위치 동기화

- AI 위치, 회전이 모든 클라이언트 화면에 주기적으로(0.1 초 간격) 반영
- 클라이언트는 보간으로 부드러운 이동 표현

### 5.3 AI 공격 이벤트 동기화

- AI 공격 발동 시 모든 클라이언트 화면에 애니메이션 및 이펙트 표시
- 근접 공격 피격 판정 타이밍은 서버 기준

### 5.4 AI 상태 동기화

- 추적, 공격, 그로기, 사망 상태 변경 시 모든 클라이언트에 반영

### 5.5 멀티플레이 스케일링 적용 시점

- 스케일링 계산은 세션 시작 시 서버에서 1회 실행
- 계산 결과(체력 배율, 스폰 수량)를 게임 상태에 저장 후 클라이언트에 반영

## 6. 보스 동기화

### 6.1 보스 HP 동기화

- 보스 HP는 서버에서 관리
- HP 변경 시 모든 클라이언트 화면의 HUD에 즉시 반영

### 6.2 보스 그로기 게이지 동기화

- 그로기 게이지는 서버 관리, 클라이언트에 반영
- 그로기 진입 / 종료 이벤트는 모든 플레이어에게 동시 반영

### 6.3 페이즈 전환 동기화

- **페이즈 전환은 서버에서 HP 임계값 감지 후 발동**
- 페이즈 전환 연출이 모든 클라이언트 화면에 동시에 재생됨

### 6.4 패턴 이벤트 동기화

- 투사체 소환, 전멸기 캐스팅 시작/종료 등 패턴 이벤트가 모든 플레이어 화면에 동시 반영

### 6.5 약점 노출 타이밍 동기화

- 약점 피격 판정 활성화/비활성화는 서버에서 처리
- 약점 노출 시각 표현이 모든 클라이언트 화면에 동시 반영

---

## 7. 월드 상호작용 동기화

### 7.1 체크포인트 동기화

- 체크포인트 활성화는 서버에서 처리 후 게임 상태에 기록
- 활성화 상태가 파티 전체에게 동시 반영

### 7.2 포탈 / 지역 이동 동기화

- 호스트가 지역 선택 시 서버에서 파티 전체 씬 전환 트리거
- 모든 파티원이 동시에 씬 전환

### 7.3 픽업 아이템 동기화

- 아이템 픽업 요청은 서버에서 처리 후 픽업 오브젝트 제거
- 제거 상태가 모든 클라이언트에 동시 반영

### 7.4 환경 오브젝트 파괴 동기화

- 파괴 이벤트는 서버에서 처리
- 파괴 상태가 모든 클라이언트에 반영 및 유지

### 7.5 보스 아레나 트리거 동기화

- 아레나 진입 확정, 장벽 활성화/해제 이벤트가 모든 플레이어에게 동시 반영

## 8. 연결 안정성 및 예외 처리

### 8.1 클라이언트 연결 해제

- 일반 접속 끊김 감지 시 해당 플레이어 제거 및 세션 지속
- 남은 파티원에게 연결 해제 알림 표시
- **호스트 플레이어 연결 끊김 시 세션 즉각 파괴 및 모든 플레이어 로비(메인 메뉴)로 강제 퇴장** (호스트 마이그레이션 기능 배제)

### 8.2 재접속 처리

- 일시적 연결 끊김 후 재접속 허용 및 상태 복원: TBD

### 8.3 이동 보정

- 클라이언트 예측 이동과 서버 위치 간 오차가 임계치(5 cm) 초과 시 보정 적용

### 8.4 신뢰도 기반 전송

- 중요 상태 변경(사망, 페이즈 전환, 아이템 지급)은 반드시 전달 보장 방식으로 전송
- 위치 동기화 등 빈도 높은 데이터는 전달 비보장 방식으로 최적화

### 8.5 타임아웃 처리

- 일정 시간(기본 30 초) 응답 없는 클라이언트 강제 해제
- 강제 해제 전 재연결 시도 알림 표시

## 12. 보상 분배 동기화

### 12.1 공유 획득

1. 플레이어가 공유 아이템 픽업 상호작용 (**1명이 상호작용을 시작(선점)하면 타인의 화면에서 프롬프트 즉시 제거**)
2. 서버에서 공유 아이템 확인
3. 파티원 전원 인벤토리에 아이템 추가 (서버 처리)
4. 픽업 오브젝트 월드 제거, 모든 플레이어에게 반영
5. 각 클라이언트에 획득 알림 UI 표시

### 12.2 개인 획득

1. 안구에 할당된 플레이어만 개인 아이템 픽업 상호작용 가능 (타인에게는 보이지 않거나 상호작용 비활성화)
2. 서버에서 개인 아이템 확인
3. 해당 플레이어 인벤토리에만 아이템 추가
4. 픽업 오브젝트 월드 제거, 모든 플레이어에게 반영
