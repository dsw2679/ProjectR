## 1. NPC 상호작용 시스템

### 1.1 거점 NPC 배치

거점 내 역할별 NPC를 배치한다. 각 NPC 근접 시 상호작용 프롬프트 표시.

| NPC | 역할 | 해금 조건 |
| --- | --- | --- |
| 상인 NPC | 무기, Mod, 소비 아이템 구매/판매 | 게임 시작 시 기본 해금 |
| 강화 NPC | 무기 / Mod 강화 처리 | TBD (특정 보스 처치 또는 기본 해금) |
| (추가 NPC 가능) | TBD | TBD |

### 1.2 NPC 대화 상호작용 흐름

1. 플레이어가 NPC 감지 범위(200 cm) 내 진입
2. 상호작용 프롬프트 표시
3. E키 입력으로 NPC 기능 UI 오픈
4. 거래 또는 강화 완료 후 UI 닫기

### 1.3 NPC 해금 상태 관리

- NPC 해금 상태는 세이브 데이터에 저장
- 해금 조건 충족 시 서버에서 해금 상태 갱신 후 모든 플레이어에게 반영
- 멀티플레이 시 호스트 세계의 NPC 해금 상태 적용

## 2. 상점 시스템

### 2.1 상점 목록 구성

상인 NPC와 상호작용 시 상점 UI 표시. 상점 목록은 상점 데이터로 관리.

| 카테고리 탭 | 표시 항목 |
| --- | --- |
| 무기 | 구매 가능한 무기 목록 |
| Mod | 구매 가능한 Mod 목록 |
| 소비 | 소비 아이템(심장, 붕대 등) 목록 |

### 2.2 상점 데이터 구조

| 항목 | 설명 |
| --- | --- |
| 상점 식별자 | 상점 고유 ID |
| 판매 아이템 ID | 해당 상점에서 판매하는 아이템 ID |
| 구매 가격 | 구매 시 필요한 재화 수량 |
| 판매(분해) 가격 | 판매 시 획득하는 재화 수량 |
| 최대 재고 수량 | -1이면 무제한 |
| 해금 조건 | 없으면 기본 노출 |

### 2.3 구매 처리 흐름

1. 상점 UI에서 아이템 선택 후 구매 버튼 클릭
2. 보유 재화 ≥ 구매 가격 확인
3. 재화 차감
4. 아이템을 플레이어 인벤토리에 추가
5. 재고 수량 감소 (재고 있는 경우)
6. 구매 확인 피드백 사운드 재생

### 2.4 판매/분해 처리

- 인벤토리에서 아이템 선택 후 판매(분해) 버튼 입력
- 재화 획득 = 판매 가격
- 인벤토리에서 아이템 제거

### 2.5 상점 진행도

- 특정 조건(보스 처치, 퀘스트 완료 등) 충족 시 새 아이템 상점에 추가
- 상점 진행도는 세이브 데이터에 저장

## 3. 재화(통화) 시스템

### 3.1 재화 종류

| 재화명 | 설명 |
| --- | --- |
| 스크랩(Scrap) | 기본 통화. 몬스터 처치, 아이템 판매 등으로 획득 |

> 재화 종류 확정: TBD. 현재 단일 재화(스크랩) 방향으로 설계.

### 3.2 재화 획득 경로

| 경로 | 획득량 |
| --- | --- |
| 일반 몬스터 처치 | 몬스터 데이터 테이블에서 정의 (예: 10 ~ 30 스크랩) |
| 보스 처치 | 보스 데이터 테이블에서 정의 (예: 200 ~ 500 스크랩) |
| 아이템 판매(분해) | 상점 데이터의 판매 가격 기준 |
| 탐색 중 발견 픽업 | 레벨 배치 재화 픽업 오브젝트 (예: 5 ~ 50 스크랩) |

### 3.3 재화 소비 경로

| 경로 | 소비량 |
| --- | --- |
| 상점 아이템 구매 | 상점 데이터의 구매 가격 기준 |
| 무기 강화 | 강화 데이터 테이블 기준 |

### 3.4 멀티플레이 재화 처리

- 재화 픽업은 개인 획득 규칙 적용 (픽업한 플레이어만 획득)
- 몬스터 처치 시 재화 획득: 파티원 전원에게 동일 금액 지급 (공유 획득 규칙)
